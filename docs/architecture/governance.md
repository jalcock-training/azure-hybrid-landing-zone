# Governance Architecture

## Overview

Governance is implemented as a modular Terraform layer that applies subscription-level policies and standards.

## Phase 1 Policies

- Allowed locations
- Required tags
- Secure-by-default configuration
- Separation of governance module from resource modules

## Phase 2 Policies (Roadmap)

- Microsoft Cloud Security Benchmark (MCSB)
- Allowed SKUs
- Diagnostic settings enforcement
- Private endpoint enforcement
- Tag enforcement
- Remediation tasks
- Compliance dashboards

## Rationale

- Ensures consistent resource configuration
- Enforces organisational standards
- Provides a foundation for future compliance automation
